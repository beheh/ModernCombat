AddAssaultTarget

Kategorie: Ziele (global)

Beschreibung

Erstellt ein neues Zielobjekt. Wird für die Spielziele Assault und Base Assault benötigt.

Syntax

bool AddAssaultTarget (id idDef, int iX, int iY, int iMaxDamage, int iTeam, string szName, int iIndex, array aSpawns, bool fNoBar);

Parameter

idDef:
Gibt an, welche Bilddefinition das Zielobjekt verwenden soll.
iX:
X-Position des neuen Zielobjekts.
iY:
Y-Position des neuen Zielobjekts (Unterkante).
iMaxDamage:
Maximalschaden, den das Zielobjekt aushält, bevor es zerstört wird.
iTeam:
Teamnummer. Diesem Team gehört das Zielobjekt. Beim Spielziel Assault muss immer das Verteidigerteam angegeben werden.
[opt] szName:
Name des neuen Zielobjekts. Wenn nicht angegeben, wird der Name von idDef verwendet.
iIndex:
Index des neuen Zielobjekts. Muss angegeben werden und sollte für jedes Team neu von 0 hochgezählt werden. Wird ein Index für ein Team doppelt verwendet, wird das alte Objekt überschrieben.
aSpawns:
Erwartet ein zwei- bzw. dreidimensionales Array. Kann bei Base Assault weggelassen werden.
[opt] fNoBar:
Wenn wahr, wird unter dem Zielobjekt keine Statusleiste angezeigt.

Anmerkung

Die Handhabung unterscheidet sich leicht zwisschen den beiden Spielzielen. Bei Assault darf nur ein einziges Team Zielobjekte erhalten. Dieses wird automatisch als Verteidigerteam eingestuft.
Anmerkung zu aSpawns: Für Base Assault wird ein zweidimensionales Array erwartet. Die erste Ebene enthält beliebig viele Spawnpunkte der Form [X, Y]. Spieler, die bei diesem Objekt einsteigen, starten zufällig an einer der Positionen. Wird nichts angegeben, starten sie an der Position des Zielobjekts. Für Assault gelten ähnliche Regeln, allerdings wird eine Team-Dimension zwischengeschoben. Das Verteidigerteam erhält den ersten Index, das Angreiferteam den zweiten. Für ein Beispiel siehe Punkt Beispiel.
Die Funktion erstellt kein Objekt des Typs idDef. Es wird lediglich ein zentrales Zielobjekt erstellt, welches das Aussehen der Definition imitiert.

Beispiel

AddAssaultTarget(WIPF, 400, 400, 250, 1, "Knuddelwipf", 0, [ [ [100, 200], [300, 400] ], [ [100, 500], [500, 100] ] ]);
Fügt ein Wipf-Zielobjekt an der Stelle 400, 400 für das Verteidigerteam Team 1 hinzu. Die Spawnpunkte sind für das Verteidigerteam zufällig 100, 200 oder 300, 400 und für das Angreiferteam 100, 500 oder 500, 100.
